Vocabulary Pt.2

Project

A project is a type of assignment, typically involving research or design, that is carefully planned to achieve a specific objective.

unit test

A unit test is the verification that a small piece of code functions and works correctly

waterfall

Waterfall is a sequential development process that flows like a waterfall through all phases of a project

software requirements

Software requirements for a system are the description of the system should do, the service or services that it provides and the constraints on its operation

Task list

A task list is a set of tasks that each vendor on a separate line with clickable checkboxes.

Milestone

A certain point of a project life that is used to measure the progress of the project and to show how close you are to your ultimate goal.

Sprint

Sprint is an arrangment of set amount of time to get coding work done

Plan

Decide on and arrange in advance

Agile

Being able to respond to change in an extremely quick manner.

Gantt chart

Project work-over a period of time, chart represents data

Maintenance branch

A maintenance branch is a branch that is used to quickly patch production releases.

backlog

an accumulation of tasks unperformed or materials not processed
